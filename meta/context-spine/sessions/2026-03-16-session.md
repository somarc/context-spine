## Summary
- Collapsed the default onboarding path into `context:setup` and `context:refresh` so first use no longer requires users to remember separate init/update/embed steps.
- Reworked bootstrap, README, and runbooks to present the lean path first and push lower-level retrieval commands into an advanced layer.
- Made bootstrap mode-aware so active sessions get previews instead of full-file dumps and stale recovery remains explicit.
- Updated the upgrade contract so older installs pick up the new lean entrypoints as safe additive helpers.

## Current State
- `package.json` now exposes `npm run context:setup` and `npm run context:refresh`.
- `scripts/context-spine/setup.sh` provides the one-command first-use path and `scripts/context-spine/refresh.sh` provides the one-command retrieval refresh path.
- Bootstrap now detects `fresh`, `recovery`, and `active` modes, shows compact previews in the active case, and skips deep QMD triage unless recovery actually needs it.
- `hot-memory.py` now picks the latest session by session date instead of file mtime, so touching an older session no longer steals the working-set slot.

## Decisions
- Make the default path one setup command and one refresh command; keep the granular retrieval commands available but demote them from the main narrative.
- Trim ceremonial onboarding guidance from the docs and replace it with a fast path that gets a repo useful quickly.
- Treat the new helper scripts as safe additive upgrade surfaces so existing installs can adopt them without clobbering project-owned entrypoints.

## Constraints / First Principles
- The lean path still needs to be explicit about what it does; simplification should remove decisions, not hide state.
- Existing projects may already rely on the low-level commands, so the old entrypoints must stay valid.
- Context Spine should become more useful by reducing ceremony, not by adding more doctrine or files.
