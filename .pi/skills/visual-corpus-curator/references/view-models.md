# View Models

These are the default visual views for a corpus.

Do not generate all of them by reflex. Generate only the views the corpus can
support with truthful data.

## 1. Capture View

Best for:

- one day
- one session
- one milestone checkpoint

Should include:

- headline
- short summary
- key metrics or checkpoints
- coherent work waves
- top surfaces changed
- next targets

This is the most important view. Build it first.

## 2. Index View

Best for:

- browsing many captures for one scope
- selecting a capture by date
- quick scan of recent work

Should include:

- one card or row per capture
- date
- headline
- summary
- key metric teaser
- links to compare or trend views when present

## 3. Compare View

Best for:

- before vs after
- adjacent work sessions
- milestone checkpoint comparisons

Should include:

- shared scope and metric basis
- changed metrics
- changed surfaces
- changed priorities or next targets
- explicit warning if the captures are not directly comparable

Avoid compare pages if the metric basis changed and no honest reconciliation is
possible.

## 4. Trend View

Best for:

- repeated captures over time
- stable metrics such as coverage, latency, or issue counts

Should include:

- time-series line or bar views
- event annotations for major refactors or milestones
- warnings where metric definition changed

Trend views should favor clarity over density. If the corpus only has two
captures, compare view is usually better than trend view.

## 5. Corpus Overview

Best for:

- multiple scopes
- executive or cross-project review

Should include:

- scope cards
- latest capture per scope
- key movement signals
- links into each scope index

Use this only after at least one scope-level corpus is already healthy.
