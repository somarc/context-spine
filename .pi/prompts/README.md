# Prompts

Place reusable prompt scaffolds here.

Good fit:

- recurring brief formats
- handoff templates
- investigation starters
- structured delegation prompts

Bad fit:

- project status
- architecture truth
- runbook content
- copied docs that will drift

Prompts should point back to the real source-of-truth surfaces instead of replacing them.

If a workflow is substantial enough to deserve a name, validation, or scripts, promote it into a skill under `../skills/` instead.

Read the fuller guidance in [../../docs/runbooks/pi-extension-points.md](../../docs/runbooks/pi-extension-points.md).
