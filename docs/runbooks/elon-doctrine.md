# E.L.O.N. Doctrine

## Goal

Keep Context Spine honest about value.

E.L.O.N. is the project doctrine for judging whether a change makes the memory system more useful and more trustworthy, or just more elaborate.

## The Doctrine

- **Evidence over aspiration**
  - Prefer verified reality over intended state, hopes, or elegant stories.
- **Legibility over lore**
  - The repo should explain itself without relying on private memory or oral tradition.
- **Optimize for utility x impact**
  - Build the thing that most improves restart quality, handoff quality, or drift resistance for the most important workflows.
- **No blind inference**
  - The next agent or maintainer should be guided toward authoritative files, verified commands, and explicit uncertainty.

## What It Is For

Use E.L.O.N. when deciding:

- whether a new memory feature should exist
- whether a note deserves promotion into a durable surface
- whether a workflow is too ceremonial
- whether a visual explainer is useful or decorative
- whether a maintenance step is pulling its weight

## What It Rejects

E.L.O.N. rejects:

- complexity without measured usefulness
- stale or aspirational documentation
- memory surfaces that look impressive but do not improve restart quality
- features that make agents sound confident while still guessing

## Questions To Ask

### Evidence over aspiration

- What pain or failure mode does this respond to?
- What command, code path, or observed drift justifies it?

### Legibility over lore

- Will a new maintainer understand the project faster because of this?
- Does it reduce the need for recaps or tribal knowledge?

### Optimize for utility x impact

- Which workflow improves?
- How often will this matter?
- Does it help one moment or every future restart?

### No blind inference

- Does this reduce ambiguity?
- Does it point to source-of-truth files or validated commands?
- Does it make uncertainty explicit?

## Preferred Outcomes

After an E.L.O.N. review, prefer one of:

- keep and strengthen
- simplify
- defer
- remove

## Relationship To Other Context Spine Surfaces

- `doctor`

- `principal-engineer-review`

- `elon-doctrine`
  - checks whether the thing is genuinely worth existing

All three are different and useful.
