# Docs Guide

This directory is split by authority, not by age alone.

## Canonical

- [../README.md](../README.md)
  Project overview and entrypoint.
- [adr/](./adr/)
  Durable architectural decisions.
- [runbooks/](./runbooks/)
  Repeatable operating knowledge.
- [templates/](./templates/)
  Starter templates for durable notes and session summaries.

## Support

- [assets/](./assets/)
  Images, rendered media, and other referenced assets.

## Drafts

- [drafts/](./drafts/)
  Work-in-progress material that should not be treated as current project truth.

## Archive

- [archive/](./archive/)
  Retired or superseded docs kept for historical context.

## Rule

If a document changes how someone should understand or operate the repo, it should live in a canonical surface or be linked from one. If it is exploratory or obsolete, move it into `drafts/` or `archive/`.
