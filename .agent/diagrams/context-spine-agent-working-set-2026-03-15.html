Context Spine · Agent Upgrade

Hot memory should be a working set, not a pile of recent edits.

The old model was mostly file recency under meta/context-spine. That is better than nothing, but still leaves agents guessing. The upgraded model points directly at the baseline, the active session, the baseline’s source-of-truth files, canonical runbooks, and the most relevant visual explainers.

Why This Matters

Blind inference usually starts when the first files opened are the wrong ones.

GPT-grade reasoning improves when the repo itself narrows the opening read set. Instead of making the agent infer the current model from scattered notes, the memory spine now gives it a deliberate sequence of trusted surfaces.

baseline first
latest intent second
source-of-truth code and docs next
visual explainers when shape matters
New Reading Path

The opening sequence is now explicit

1. Baseline note Open the durable model of the repo before looking at fresh churn.
2. Latest session Recover active work, current intent, and what was last verified.
3. Source of truth Open the exact repo files named by the baseline instead of searching broadly.
4. Canonical docs Read the runbooks that control the current operating path.
5. Visual explainers Use diagrams for system shape, upgrade paths, and skill orchestration.

Session Contract

Session notes now carry execution state, not just prose

Branch / HEAD / worktree
Lets the next agent or maintainer restart from the actual code state instead of assuming the repo was clean or linear.
Last meaningful verification
Captures the command that last proved something important so the next session can anchor on evidence quickly.
QMD triage still required
The retrieval loop remains first-class; the session does not replace search, it makes search sharper.
