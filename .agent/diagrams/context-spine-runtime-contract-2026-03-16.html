Context Spine · Runtime Contract Upgrade · 6e590f6

Make the memory system prove its own runtime.

This change set made Context Spine more like an auditable runtime and less like a pile of helpful scripts. The core shift is simple: the repo now has a versioned runtime contract, deterministic skill sync validation, and structured run records for the maintenance commands that keep trust surfaces current.

Skill digests

What changed materially

doctor, hot-memory, rollout, and upgrade now leave an explicit trace. Skill install and verification now check normalized directory digests, so repo skill source and installed Codex copies can be proven identical.

Why it matters

Context Spine already cared about evidence. This makes the runtime itself evidence-bearing: versioned, inspectable, and able to answer “what exactly ran?” without relying on chat memory.

Before / After

Before

  • Skill sync validated syntax, but not source-to-installed equality.
  • No single runtime manifest described version, compatibility, or shared runtime surfaces.
  • Maintenance commands generated aids, but did not emit a durable run identity or structured state record.

After

  • runtime-manifest.json defines the shared runtime contract.
  • context:skill:verify-installed proves installed skill copies match repo source digests.
  • Maintenance commands emit run IDs and write JSON records under meta/context-spine/runs/.
Flow
1. Contract

Runtime manifest becomes the shared truth

scripts/context-spine/runtime-manifest.json declares the runtime version, compatibility expectations, generated aids, skill names, and core runtime files.

2. Execution

Maintenance commands start with a run handle

doctor, hot-memory, rollout, and upgrade create a run ID immediately and attach runtime metadata before they emit reports.

3. Verification

Skill install now compares digests

Source skill directories under .pi/skills/ are compared to installed copies in ~/.codex/skills/ using normalized directory digests.

4. Evidence

Generated artifacts become traceable

Reports still exist, but now they can be tied back to a structured JSON run record, giving Context Spine a better answer to “what happened, when, and under which contract?”

The Three Additions
Addition A

Versioned runtime manifest

The new manifest is the boilerplate-side contract for shared runtime surfaces. It reduces drift between doctor, upgrade, gitignore policy, and future automation by keeping the runtime description in one place instead of spreading it across scripts.

  • runtime version
  • compatibility metadata
  • runtime files
  • generated aid classes
Addition B

Source / installed digest verification

This closes a real trust gap. Syntax-only validation could not prove that the repo skill source and the installed Codex skill copy were actually the same. Digest comparison now gives a binary answer.

  • validate-codex-skill.py --compare
  • install-codex-skill.sh --verify-installed
  • npm run context:skill:verify-installed
Addition C

Run IDs and structured run state

The maintenance layer now carries its own execution trail. Each run gets a stable ID, summary, status, runtime version, and artifact list. That makes generated aids auditable instead of orphaned.

  • JSON records under meta/context-spine/runs/
  • run ID emitted in terminal output
  • artifacts linked in structured state
Operational Surfaces

Commands that now leave structured traces

context:doctor Loads config and runtime manifest, emits a run ID, writes report plus run record.
hot-memory.py Generates the working-set index and records the run metadata that produced it.
rollout.py Aggregates multi-repo status and records the run contract for the rollout report.
upgrade.py Evaluates upgrade-safe changes under a versioned contract and records the run summary.

Key files added or materially changed

scripts/context-spine/runtime-manifest.json New versioned runtime contract.
scripts/context-spine/runtime_manifest.py Manifest loader plus normalized file and directory digests.
scripts/context-spine/run_state.py Run handle creation and structured JSON persistence.
scripts/context-spine/install-codex-skill.sh Now verifies installed skill copies against source digests.
scripts/context-spine/validate-codex-skill.py Now validates and compares skill directory digests.
