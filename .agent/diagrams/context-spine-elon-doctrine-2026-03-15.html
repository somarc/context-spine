Context Spine · Doctrine

E.L.O.N. is the test for whether the memory system deserves to exist.

This doctrine turns a vague product taste into an operating filter. If a feature, workflow, note, or diagram does not improve evidence, legibility, utility, or inference quality, it should be simplified, deferred, or removed.

Doctrine

The four letters

Evidence over aspiration
Legibility over lore
Optimize for utility x impact
No blind inference
E

Evidence over aspiration

The repo should reflect what is true, not what is planned, imagined, or aesthetically satisfying.

  • commands beat claims
  • verified docs beat aspirational docs
  • doctor and evidence surfaces matter
L

Legibility over lore

Another human or agent should be able to restart without depending on private memory or oral tradition.

  • baseline and session should open the repo correctly
  • visual explainers are justified when they reduce orientation cost
  • lore is a failure mode
O

Optimize for utility x impact

Context Spine should help the most important workflows for the most meaningful restarts, handoffs, and audits.

  • cold-start recovery
  • mid-project resume
  • handoff quality
  • truth maintenance
N

No blind inference

The next agent should be guided toward authority and explicit uncertainty instead of sounding confident while guessing.

  • working set beats recency dump
  • source_of_truth must matter
  • uncertainty should be surfaced, not hidden
Why This Fits Context Spine

It turns “is this clever?” into “does this reduce entropy?”

In the LLM-assisted world, endless creation is easy. The hard part is building only the layers that improve restart quality, correctness, and handoff. E.L.O.N. is the anti-ornament doctrine.

What It Rejects

Keep

  • working-set hot memory
  • doctor and upgrade
  • skills that add orchestration
  • diagrams that reduce orientation time

Reject

  • ceremonial note sprawl
  • features without measured value
  • beautiful but disconnected explainers
  • confidence without source grounding
Preferred Outcome

Every meaningful change should end in one of four judgments

keep and strengthen
simplify
