Context Spine

A reusable intelligence layer for projects that need working memory, retrieval, durable notes, and evidence discipline from day one.

Context Spine exists because rich project context does not appear by accident. It compounds when a project has a read loop, a write loop, and explicit places for memory, retrieval, visual reading, and extension.

The Five Layers

Context Spine carries the Oak memory model into a reusable repository. Each layer serves a different time horizon, and the system only works well when they stay linked.

Layer 1
Highest truth

Live project reality

Code, tests, scripts, logs, command output, and runtime behavior. This is what has actually been delivered.

source codetestslogsruntime
Layer 2
Short horizon

Repo-local working memory

`meta/context-spine` holds observations, session summaries, evidence packs, hot memory, and memory scoring close to the code.

observations/sessions/evidence-packs/scorecard
Layer 3
Retrieval fabric

QMD-backed search

Retrieval turns repo memory, docs, and vault notes into one query plane instead of separate folders and hope.

init-qmd.shqmd-quick.shmem-search.py
Layer 4
Long horizon

Durable linked understanding

External notes carry deep dives, audits, baselines, and hubs that should survive branch churn and session resets.

hub notesauditsbaselinessources
Layer 5
Extensible

Bounded delegation and explainers

Delegation stays useful when it emits evidence-shaped output, and visual explainers become a normal reading surface instead of optional polish.

evidence packs.pi/.agent/diagrams/

The key design move is that Context Spine does not try to replace Git, QMD, notes, or agent runtimes. It binds them into one operating loop.

Read And Write Loops

The template becomes valuable only when retrieval and synthesis reinforce each other. That is what made Oak feel natural, and that is what this repo is trying to preserve.

Read loop

Hydrate the session from memory and retrieval before diving into the codebase.

1
BootstrapRun the bootstrap script to surface prerequisites, hot memory, recent explainers, and the latest session.
2
Query retrievalUse `qmd` through the quick search or targeted searches before broad scans.
3
Read the explainerIf a subsystem already has a visual explainer, use it as the fast reading surface.
4
Re-anchor in codeReturn to code, tests, and commands before making any decision that matters.

Write loop

Convert raw work into durable memory that the next session can actually retrieve.

1
Start from evidenceBranch diffs, logs, tests, and commands produce the raw truth.
2
Capture locallyWrite session notes, observations, or evidence packs under `meta/context-spine`.
3
Synthesize durablyUpdate a durable note and, when helpful, an explainer under `.agent/diagrams/`.
4
Refresh retrievalRun `qmd update` and `qmd embed` so the new understanding becomes searchable.

Repo Shape

The repository is intentionally small. It bootstraps the loop without pretending to be the application itself.

Contract

`AGENTS.md`, ADRs, runbooks, and note templates define the working rules and the minimum durable output shape.

AGENTS.md docs/adr/ docs/runbooks/ docs/templates/

Memory tooling

The core scripts create sessions, log observations, refresh hot memory, score memory quality, and bootstrap retrieval.

scripts/context-spine/ bootstrap.sh init-qmd.sh mem-session.py mem-log.py hot-memory.py mem-score.py

Extension surfaces

Projects can add local skills, prompts, evidence adapters, and visual explainers without breaking the spine.

Prerequisites And Growth

Context Spine stands on the shoulders of other tools. That is a feature, not a weakness. The template should remain thin and composable.

Prerequisites

Git, Python 3, a POSIX shell, and preferably QMD plus an external durable note system.

gitpython3bash/zshqmd

High-value extras

Local skills, Ollama, tmux, CI, and a reliable visual explainer workflow all make the system compounding instead of static.

.piollamatmuxCI

Growth rule

Add adapters and surfaces. Do not replace the loop. If a new capability does not improve retrieval, evidence, or durable understanding, it probably does not belong in the spine.
