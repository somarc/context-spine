Context Spine · Skill Stack

Give the memory loop operators, not just notes.

The core memory system already had baseline notes, sessions, doctor, and upgrade. The new skill layer makes that system easier to run deliberately: maintain the spine, promote recent work into the right durable surface, and assess multiple repos without reinventing the operating pattern every time.

What Changed

The new layer is orchestration.

These additions do not replace the scripts. They wrap them in named judgment paths so an operator or agent can ask the right question quickly: is this repo healthy?, what deserves promotion?, which local repos should I fix first?

script truth stays in doctor.py / upgrade.py
skills add orchestration and triage
rollout stays small-batch and local
Operating Loop

How the improved memory system now runs

1. Inspect the spine Run bootstrap, doctor, and score so the repo says whether the memory layer still maps to project truth.
2. Interpret the result Use the maintenance skill to decide what is healthy, what is drifting, and what deserves immediate repair.
3. Promote only the right truths Use memory promotion to turn rolling work into an ADR, runbook, baseline note, evidence pack, or diagram.
4. Roll forward across repos Use multi-repo rollout when a handful of local projects need the same hygiene and upgrade pass.
Skill 1

Context Spine Maintenance

Runs the core maintenance loop and tells you what to fix first.

  • prefers repo wrappers
  • anchors on doctor before opinion
  • treats score as secondary signal
Skill 2

Memory Promotion

Prevents memory sprawl by deciding what should become durable and where it belongs.

  • baseline note vs ADR vs runbook
  • evidence pack vs diagram vs session-only
  • smallest durable surface wins
Skill 3

Multi Repo Rollout

Assesses a small fleet of local repos and ranks them by urgency.

  • doctor counts per repo
  • upgrade-safe gaps vs merge-review drift
  • one rollout report instead of repeated guesswork
Use Map

Which surface answers which question?

“Is this repo healthy?”
Run context:doctor directly, or use $context-spine-maintenance when you also want prioritization.
“What should become durable?”
Use $memory-promotion after a feature, audit, investigation, or architectural turn.
“Which repos need attention first?”
Run context:rollout or use $multi-repo-rollout to compare a small local fleet.
“Should doctor itself be a skill?”
No. The script remains the deterministic source of truth. Skills are the orchestration layer around it.
Design Rules

Why this fits GPT-5.4 Extra High better

  • less prompt reconstruction and more named workflows
  • clear split between deterministic checks and human-style judgment
  • stronger repo-to-repo reuse without pretending every project is identical
  • memory stays curated instead of turning into a permanent dump of session residue
  • operators get an upgrade path instead of “reinstall and hope”
